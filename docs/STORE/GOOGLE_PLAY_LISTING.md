## 3. Graphic Assets

### 3.1 App Icon
- **Size:** 512x512 px
- **Format:** 32-bit PNG with alpha channel
- **Requirements:**
  - Clear, simple design recognizable at small sizes
  - No text overlays (icon should be symbolic)
  - Must not contain explicit nudity
  - Should represent brand identity
  - Safe for all audiences (icon is public-facing)

### 3.2 Feature Graphic
- **Size:** 1024x500 px
- **Format:** JPG or 24-bit PNG (no alpha)
- **Requirements:**
  - Eye-catching design for store listing header
  - Can include app name and tagline
  - Must not show explicit content
  - Should highlight key features visually
  - Safe-for-work imagery

### 3.3 Screenshots
- **Minimum:** 2 screenshots
- **Recommended:** 8 screenshots (to show full feature set)
- **Size:** Phone: 1080x1920 to 1080x2340 px (or higher)
- **Format:** JPG or 24-bit PNG

**Screenshot Order & Content:**
1. **Welcome/Onboarding** ‚Äî First impression of app design
2. **Discovery Feed** ‚Äî Show profile browsing/swiping
3. **Profile Example** ‚Äî Well-designed profile (blurred faces if needed)
4. **Chat Interface** ‚Äî Clean chat UI (no explicit messages)

6. **Wallet/Tokens** ‚Äî Token packages display
7. **Safety Features** ‚Äî Verification or safety UI

**Guidelines for Screenshots:**
